# Moltbot Prompt 行文风格观察（可学习点）

> 生成时间: 2026-02-01
> 来源: D:\Project\moltbot 的系统 prompt 与模板文件

## 关键风格特征
1. **短句 + 强动词**
   - 大量命令式短句，减少歧义。
   - 例：Do/Don't/Only/Always/Before。

2. **规则与边界单独成段**
   - "Rules" / "What You DON'T Do" / "Safety" 单独列出。
   - 便于模型快速锁定行为边界。

3. **明确触发条件（阈值化）**
   - HEARTBEAT_OK / NO_REPLY 等强制触发点。
   - “当…时/如果…则”显式定义。

4. **例子与反例并置**
   - 正确/错误示范减少输出偏差。
   - 特别常见于工具与输出协议部分。

5. **段落分工清晰**
   - Tooling/Memory/Docs/Messaging/Voice 等独立段落。
   - 避免一个段落里混合多个主题。

6. **人味与硬规则分层**
   - 模板文件更“对话感”。
   - 系统 prompt 更“合同式”。
   - 降低冲突与风格漂移。

7. **决策树语气**
   - 大量 Before/If/When/Otherwise。
   - 引导模型走明确分支。

8. **术语一致性**
   - main session / subagent / heartbeat 等术语统一。
   - 认知负担更低。

## 可直接迁移到 Mimikit 的写作建议
- 以“规则块 + 例子块 + 触发条件块”组合描述。
- 强化“只/必须/禁止”等硬边界语言。
- 对每个工具或输出格式提供最小正确示例。
- 对多输入/多工具调用明确顺序与数量规则。
