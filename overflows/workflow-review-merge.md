# Workflow: review and merge

## Purpose
- Record the local workflow for review, commit, sync, and merge.

## Steps
1. Run the review-code-changes skill and ensure it passes.
2. Create a commit with an auto-written message.
3. Pull the latest main into the current branch.
4. Resolve conflicts in the current branch, if any.
5. Merge into main.
6. Keep the current branch and worktree for future use.
